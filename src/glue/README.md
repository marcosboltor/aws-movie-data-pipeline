# AWS Glue

This folder contains reference files related to AWS Glue usage in the project.

In this project, AWS Glue is used as a metadata cataloging service through Glue Data Catalog and Glue Crawler. It is not used as an ETL processing engine.

Data transformations are performed by AWS Lambda and Amazon Athena.

## Files

- `crawler_config.md`: documents the configuration of the Glue Crawler used to detect the Silver layer schema and partitions.
- `database_schema.md`: documents the valid tables registered in the Glue Data Catalog.
- `repair_and_describe.sql`: contains Athena SQL queries used to repair partitions, inspect schemas and validate Silver and Gold tables.

## Glue usage in the project

AWS Glue is used for:

- Registering the Silver table in the Glue Data Catalog.
- Detecting schema and partitions from Parquet files stored in S3 Silver.
- Allowing Amazon Athena to query S3 files as SQL tables.
- Storing metadata for the Gold tables created through Athena CTAS queries.

## Valid catalog tables

The valid tables for the final project are:

- `2silver`
- `gold_performance_genero`
- `gold_ranking_peliculas`
- `gold_peliculas_sobreexpuestas`
- `gold_tendencia_generos`

## Ignored tables

The following tables were detected by the crawler but are not part of the final analytical model:

- `23bf1f627cf741529b801c552fd8f488_snappy_parquet`
- `50ab8e2b8eaa468984596f563b7c2ec4_snappy_parquet`
- `month_05`

These tables correspond to individual Parquet files or intermediate partition folders and should not be used for analysis.

## Notes

The Glue Crawler is focused on the Silver layer. The Gold tables are generated by Athena CTAS queries executed from the `silver_tmdb_to_gold` Lambda.
